# Quiz Practice Page

A standalone interactive HTML page for reviewing multiple-choice, true/false, and fill-in-the-blank questions. No server or backend required -- open it directly in any browser.

## Quick Start

Visit the hosted page:

[https://foweh.github.io/marx-quiz-practice/](https://foweh.github.io/marx-quiz-practice/)

Or download `index.html` and open it locally.

## Modes

- **Practice** -- Select an option to immediately see the correct answer and explanation. Wrong answers are saved automatically.
- **Exam** -- Answer all questions before submitting. Score and time are recorded permanently.
- **Flashcard** -- Tap the card to flip between question and answer for rapid review.

## Features

- One question at a time with keyboard navigation (arrow keys)
- Question grid for quick jumping and progress tracking
- Question type badges: single-choice, multiple-choice, true/false, fill-in-the-blank
- Multiple-choice supports selecting more than one option with a confirm button
- True/false questions auto-generate the two buttons when needed
- Fill-in-the-blank questions display the correct number of input fields
- Search bar to filter questions by keyword (stem, options, or knowledge points)
- Section filter to focus on a specific chapter
- Wrong-answer book persists in localStorage, with one-click review or re-exam
- Favorites collection with star button on each question
- Shuffle button to randomize question order
- Dark mode toggle (preference saved)
- Responsive layout for mobile and desktop
- All history and settings stored in your browser, nothing sent anywhere

## Data Persistence

Everything is stored in your browser's localStorage. No data is ever uploaded or shared.

- Wrong-answer book
- Favorites
- Exam score history (date, score, section, time spent)
- Manually filled answers
- Dark mode preference

Use the "Clear" buttons in the corresponding panels to remove saved data.

## How It Works

The page is a single self-contained HTML file. All question data is embedded as inline JSON in the page source. JavaScript renders the interactive UI and manages state entirely on the client.

## Build

No build step is needed. The HTML page is generated by a userscript that runs on a compatible course page. The userscript:

1. Scans the page structure for quiz sections
2. Fetches question data via API
3. Renders the questions into an interactive HTML file

To use the userscript, install a userscript manager (such as Tampermonkey or Violentmonkey) and add `quiz-extractor.user.js`.
